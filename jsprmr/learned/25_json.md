# 25. JSON

## このページで学ぶこと

- JSONはJavaScriptのオブジェクトリテラルをベースにした軽量なデータフォーマットであり、JavaScriptの構文とは一部異なること
- `JSON`オブジェクトの`JSON.parse`静的メソッドで、JSON形式の文字列をJavaScriptの値へ変換できること
- `JSON`オブジェクトの`JSON.stringify`静的メソッドで、JavaScriptの値をJSON形式の文字列へ変換できること
- `JSON.stringify`の`replacer`引数と`space`引数で、変換するプロパティや出力のインデントを制御できること
- JSONにシリアライズできない値や、シリアライズ時に例外が発生する値があること
- オブジェクトの`toJSON`メソッドが`JSON.stringify`の変換結果に使われること

## 課題で要求してよい知識

- JSONはJavaScript Object Notationの略で、JavaScriptのオブジェクトリテラルをベースにしたデータフォーマットであると説明できる
- JSONではオブジェクトのキーをダブルクォートで囲む必要があることを説明できる
- JSONでは小数点から始まる数値リテラルや、先頭がゼロから始まる数値リテラルを使えないことを説明できる
- JSONで表現できる文字列、数値、真偽値、`null`、配列、オブジェクトを組み合わせたJSON文字列を読解できる
- `JSON.parse`でJSON形式の文字列をJavaScriptの値へ変換できる
- JSON文字列がオブジェクトを表す場合は`JSON.parse`の返り値がオブジェクトになり、配列を表す場合は配列になることを説明できる
- JSONとしてパースできない文字列を`JSON.parse`に渡すと例外が投げられることを説明できる
- 外部から受け取ったJSON文字列を扱う場面では、`try...catch`で`JSON.parse`の例外を処理できる
- `JSON.stringify`でJavaScriptの値をJSON形式の文字列へ変換できる
- `JSON.stringify`の第二引数に関数を渡し、キーと値を受け取って返り値でシリアライズ結果を制御できる
- `JSON.stringify`の`replacer`関数が`undefined`を返したプロパティは、変換後のJSONに含まれないことを説明できる
- `JSON.stringify`の第二引数に配列を渡すと、その配列に含まれる名前のプロパティだけが変換されることを説明できる
- `JSON.stringify`の第三引数に数値を渡すとその数のスペースで、文字列を渡すとその文字列でインデントされたJSON文字列を得られることを説明できる
- `space`引数だけを指定したい場合、`replacer`引数には`null`を渡して省略するのが一般的であることを説明できる
- 文字列、数値、真偽値、`null`、配列、オブジェクトが`JSON.stringify`で対応するJSONの値へ変換されることを説明できる
- オブジェクトのプロパティ値が関数、`undefined`、`Symbol`の場合、そのプロパティは`JSON.stringify`の結果に含まれないことを説明できる
- 配列の要素が関数、`undefined`、`Symbol`の場合、その要素は`JSON.stringify`で`null`に変換されることを説明できる
- キーが`Symbol`であるプロパティは`JSON.stringify`のシリアライズ対象外になることを説明できる
- `RegExp`、`Map`、`Set`のインスタンスは列挙可能なプロパティを持たないため、`JSON.stringify`では空のオブジェクトとして変換されることを説明できる
- `BigInt`を`JSON.stringify`でシリアライズしようとすると例外が発生することを説明できる
- 循環参照を持つオブジェクトを`JSON.stringify`でシリアライズしようとすると例外が発生することを説明できる
- `JSON.stringify`も失敗することがあるため、必要に応じて`try...catch`で例外を処理できる
- オブジェクトが`toJSON`メソッドを持つ場合、`JSON.stringify`は既定の変換ではなく`toJSON`メソッドの返り値を使うことを説明できる
- `toJSON`メソッドは、`JSON.stringify`の引数に直接渡されたオブジェクトだけでなく、プロパティとして登場したオブジェクトにも再帰的に使われることを説明できる

## 前提として使う既習知識

- オブジェクトリテラルと配列リテラルを読み書きできる
- 文字列、数値、真偽値、`null`、`undefined`、`Symbol`、`BigInt`などの基本的な値を区別できる
- オブジェクトのプロパティを読み書きできる
- 関数やアロー関数を定義し、引数と返り値を扱える
- `try...catch`で例外を捕捉できる
- `Map`と`Set`のインスタンスを作成できる

## 課題で要求しない知識（必要な場合のみ）

- `JSON.parse`の第二引数であるreviver引数は要求しない
- ECMA-404やjson.orgに書かれたJSON文法の細部は要求しない
- JSONを使ったHTTP通信、ファイル入出力、ブラウザストレージ、DOM操作は要求しない
- `JSON.stringify`の仕様上の特殊ケースを、このページで扱った値以外まで網羅して要求しない

## 課題で扱う場合の注意

- JSON文字列の比較をする課題では、空白やインデントの有無を採点対象にするかを明示する
- JSON文字列としての完全一致が不要な課題では、`JSON.parse`した結果の値で比較できるようにする
- `replacer`引数を使う課題では、関数を使うのか配列を使うのかを課題文で明示する
- 例外処理を扱う課題では、エラーメッセージの完全一致ではなく、例外が発生する原因や`try...catch`で処理していることを評価する
