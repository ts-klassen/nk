# 17. ラッパーオブジェクト

## このページで学ぶこと

- プリミティブ型の値が、対応するオブジェクトのメソッドを呼び出せる仕組みとしてラッパーオブジェクトを扱う
- 真偽値、数値、BigInt、文字列、シンボルには対応するラッパーオブジェクトがあり、`undefined` と `null` には対応するラッパーオブジェクトがないことを扱う
- プリミティブ型の値へプロパティアクセスすると、自動的に対応するラッパーオブジェクトへ変換されることを扱う
- 明示的に作成したラッパーオブジェクトから `valueOf` メソッドでプリミティブ型の値を取り出せることを扱う
- プリミティブ型のデータには、ラッパーオブジェクトではなくリテラルを使うことを扱う

## 課題で要求してよい知識

- プリミティブ型の文字列は `String` オブジェクトのインスタンスではないが、`String` オブジェクトのインスタンスメソッドを呼び出せることを説明できる
- `Boolean`、`Number`、`BigInt`、`String`、`Symbol` が、それぞれ真偽値、数値、BigInt、文字列、シンボルに対応するラッパーオブジェクトであることを対応づけられる
- `undefined` と `null` には対応するラッパーオブジェクトがないことを説明できる
- `new String("文字列")` のように、プリミティブ型の値を包んだラッパーオブジェクトを作成できる
- `typeof` 演算子でプリミティブ型の文字列を評価すると `"string"` になり、`String` のラッパーオブジェクトを評価すると `"object"` になることを説明できる
- プリミティブ型の値に対してプロパティアクセスやメソッド呼び出しをすると、自動的に対応するラッパーオブジェクトへ変換されることを説明できる
- プリミティブ型の文字列で `toUpperCase` などの `String` のインスタンスメソッドを呼び出せる理由を、ラッパーオブジェクトへの自動変換と関連づけて説明できる
- 明示的に作成したラッパーオブジェクトに `valueOf` メソッドを呼び出して、包まれているプリミティブ型の値を取り出せる
- プリミティブ型のデータにはラッパーオブジェクトではなくリテラルを使うべき理由を説明できる
- 「JavaScriptはすべてがオブジェクトである」という説明は正確ではなく、プリミティブ型はオブジェクトではないがラッパーオブジェクトによってオブジェクトのように見えることを説明できる

## 前提として使う既習知識

- プリミティブ型とオブジェクトを区別できる
- 文字列、真偽値、数値、BigInt、シンボル、`undefined`、`null` のリテラルや値を扱える
- `typeof` 演算子で値の型を確認できる
- オブジェクトのプロパティアクセスとメソッド呼び出しができる
- `new` を使ってコンストラクタからインスタンスを作成できる
- `String` オブジェクトの基本的なメソッドを呼び出せる

## 課題で要求しない知識（必要な場合のみ）

- ECMAScript仕様における内部操作やアルゴリズムの詳細は要求しない
- ラッパーオブジェクトのプロトタイプチェーンや継承構造の詳細は要求しない
- `Object()` を使ったラッパーオブジェクトの作成方法は要求しない
- `Symbol` や `BigInt` のラッパーオブジェクトを明示的に作成する方法は要求しない

## 課題で扱う場合の注意

- ラッパーオブジェクトの明示的な作成は仕組みの説明に限り、通常の実装ではリテラルを使う前提にする
- `typeof` の結果を問う場合は、プリミティブ型の値とラッパーオブジェクトのどちらを評価しているかを課題文で明示する
- ボックス化やアンボクシングという用語は本文の脚注で紹介されるため、用語の暗記ではなく自動変換の仕組みを説明できることを評価する
