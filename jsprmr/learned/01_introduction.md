# 01. JavaScriptとは

## このページで学ぶこと

- JavaScriptが主にウェブブラウザで使われ、Node.js、デスクトップアプリ、スマートフォンアプリ、IoTデバイスなど幅広い環境でも使われること
- ECMAScriptが実行環境に共通するJavaScriptの仕様であり、JavaScriptには実行環境ごとの固有機能も含まれること
- JavaScriptの言語的な特徴として、大文字小文字の区別、予約語、文とセミコロン、空白文字、strict mode、ScriptとModuleの実行コンテキストを扱うこと
- ECMAScriptは毎年更新される一方で、後方互換性が重視されていること

## 課題で要求してよい知識

- JavaScriptがウェブブラウザ上の表示変更やサーバーとの通信に使われ、ウェブアプリの実現に関わることを説明できる
- JavaScriptがブラウザ以外にもNode.js、デスクトップアプリ、スマートフォンアプリ、IoTデバイスなどで使われることを説明できる
- ECMAScriptは実行環境に共通する動作を定める仕様であり、JavaScriptはECMAScriptと実行環境固有の機能を含む範囲だと区別できる
- 実行環境によって必要な機能が異なるため、ブラウザやNode.jsなどがそれぞれ固有の機能を定義することを説明できる
- ECMAScriptで定義された機能を学ぶことが、複数の実行環境に共通する基本部分の学習になることを説明できる
- ECMAScriptは毎年更新され、新しい文法や機能が追加されるため、実行環境によって対応するECMAScriptのバージョンが異なる場合があることを説明できる
- ES2015はECMAScriptの大きな更新であり、この書籍がES2015に基づいた内容であることを説明できる
- JavaScriptのオブジェクトには、ECMAScriptが定義するもの、実行環境が定義するもの、ユーザーが定義するものがあると分類できる
- JavaScriptでは大文字と小文字が区別され、`name`と`NAME`は別々の名前として扱われることを説明できる
- 大文字で始めるなどの命名規則自体に言語上の意味はなく、大文字小文字の違いは単に別々の名前として扱われることを説明できる
- JavaScriptには特別な意味を持つ予約語があり、`const`のような予約語を変数名や関数名として宣言できないことを説明できる
- JavaScriptの文はセミコロンで区切られ、暗黙的なセミコロン挿入に頼らずセミコロンを書く方針を説明できる
- スペースやタブなどの空白文字は、本文の例のような式や文の間に複数置いても動作を変えないことを説明できる
- `"use strict";`をファイルまたは関数の先頭に書くと、そのスコープのコードがstrict modeで実行されることを説明できる
- strict modeでは古く安全でない構文や機能の一部が禁止され、明らかな問題を含むコードが例外になることを説明できる
- strict modeでは、`const`などの宣言キーワードなしに変数へ代入しようとすると`ReferenceError`になることを説明できる
- JavaScriptの実行コンテキストにはScriptとModuleがあり、Scriptは多くの実行環境でデフォルト、ModuleはES2015で導入されデフォルトでstrict modeになることを説明できる
- モジュールの機能はModuleの実行コンテキストでしか利用できないことを説明できる
- ECMAScriptでは後方互換性が重視され、過去に書いたJavaScriptコードが動かなくなる変更はほとんど入らないことを説明できる

## 前提として使う既習知識

- 特になし。このページは第一部の導入であり、JavaScriptの文法知識を前提にしない

## 課題で要求しない知識（必要な場合のみ）

- 変数宣言、関数宣言、オブジェクト、スコープ、モジュール構文の具体的な書き方や詳細な仕様は要求しない
- DOM API、ブラウザのUI操作、サーバー通信、Node.js APIなど、実行環境固有機能の実装は要求しない
- ECMAScriptの仕様策定プロセスや、各ECMAScriptバージョンの差分の詳細は要求しない
- 自動セミコロン挿入の詳細な規則や、strict modeで禁止される構文の網羅的な知識は要求しない
- `eval`、`with`、グローバル変数の詳細な挙動は要求しない

## 課題で扱う場合の注意

- このページは導入なので、課題は用語の区別、コード断片の読み取り、挙動の説明を中心にし、後続章の構文知識を前提にしない
- `const`を含むコード例を扱う場合でも、変数宣言そのものの詳細な理解は採点対象にしない
- strict modeに関する課題では、エラーメッセージの完全一致ではなく、strict modeで問題になる理由や`ReferenceError`になることを評価する
- ブラウザとNode.jsなどの実行環境を比較する場合は、固有APIの使い方ではなく、共通部分と固有部分の区別を評価する
