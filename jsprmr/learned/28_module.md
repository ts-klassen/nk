# 28. ECMAScriptモジュール

## このページで学ぶこと

- ECMAScriptモジュールは、JavaScriptファイル単位で変数や関数などをまとめ、エクスポートとインポートで別のモジュールから利用できる仕組みである
- ECMAScriptモジュールの基本構文として、名前つきエクスポート／インポート、デフォルトエクスポート／インポート、エイリアスを扱う
- 再エクスポート、名前空間インポート、副作用のためのインポート、Dynamic Import、インポート属性を扱う
- ウェブブラウザでECMAScriptモジュールを実行するには、`script`要素に`type="module"`を指定する必要がある

## 課題で要求してよい知識

- 1つのECMAScriptモジュールが1つのJavaScriptファイルに対応することを説明できる
- モジュールを使う目的として、保守性、名前空間の分離、再利用性を説明できる
- モジュールから変数や関数などをエクスポートし、別のモジュールでインポートして利用できることを説明できる
- `export { name }`構文を使って、宣言済みの変数や関数を名前つきエクスポートできる
- `export function name() {}`のように、宣言と同時に名前つきエクスポートできる
- `import { foo, bar } from "./my-module.js"`のように、名前つきエクスポートされた値を名前を指定してインポートできる
- 複数の名前つきエクスポートや名前つきインポートを、カンマで区切って指定できる
- `as`を使って、名前つきエクスポートや名前つきインポートに別名をつけられる
- デフォルトエクスポートは、モジュールごとに1つだけエクスポートできる特殊なエクスポートであることを説明できる
- `export default 式`を使って、式の評価結果をデフォルトエクスポートできる
- 関数やクラスは、宣言と同時にデフォルトエクスポートでき、そのとき名前を省略できることを説明できる
- 変数宣言は`export default const foo = ...`のように宣言と同時にデフォルトエクスポートできないことを説明できる
- `import myModule from "./my-module.js"`のように、デフォルトエクスポートに任意の名前をつけてインポートできる
- デフォルトエクスポートは`default`という固有名の名前つきエクスポートとして扱えることを説明できる
- `export { foo as default }`で名前つきエクスポートに`default`の別名をつけ、デフォルトエクスポートできる
- `import { default as name } from "./my-module.js"`でデフォルトエクスポートを名前つきインポートの構文でインポートできる
- デフォルトインポートと名前つきインポートを、`import defaultName, { named } from "./my-module.js"`のように同時に書ける
- エクスポートされていない値はインポートできず、指定したエクスポートが存在しない場合はパースエラーになることを説明できる
- `export ... from "./my-module.js"`構文で、別のモジュールからインポートしたものを再エクスポートできる
- すべての名前つきエクスポート、選択した名前つきエクスポート、エイリアスをつけたエクスポート、デフォルトエクスポートを再エクスポートする構文を読める
- `import * as moduleName from "./my-module.js"`で、すべての名前つきエクスポートを名前空間オブジェクトとしてまとめてインポートできる
- 名前空間インポートで作られたオブジェクトのプロパティから、名前つきエクスポートや`default`にアクセスできる
- `import "./side-effects.js"`のように、何もインポートせずに指定したモジュールのグローバルコードだけを実行できる
- 通常の`import`文は静的に解析されるため、条件分岐の中でモジュールをインポートできないことを説明できる
- `import("./module.js")`のDynamic Importを使って、条件分岐や必要になったタイミングで動的にモジュールをインポートできる
- Dynamic ImportがPromiseを返し、`.then`メソッドや`await`式と組み合わせて結果を扱えることを説明できる
- Dynamic Importで読み込んだモジュールから、名前つきエクスポートやデフォルトエクスポートにプロパティとしてアクセスできる
- Top-Level awaitを使うと、モジュールのトップレベルでDynamic Importの結果を`await`できることを説明できる
- インポート属性を使うと、インポートするモジュールに関する追加の属性情報を指定できることを説明できる
- 静的なインポートで`with { type: "json" }`を指定し、JSONモジュールとして読み込む構文を読める
- Dynamic Importの第二引数に`{ with: { type: "json" } }`を指定する構文を読める
- インポート属性でサポートされる`type`は実行環境によって異なることを説明できる
- ウェブブラウザでは、`<script type="module" src="./my-module.js"></script>`でJavaScriptファイルをECMAScriptモジュールとして読み込める
- インラインの`script`要素でも、`type="module"`を指定するとECMAScriptモジュールとして扱われることを説明できる
- `type="module"`を指定しない通常のスクリプトでは、`import`文や`export`文を使うと構文エラーになることを説明できる
- ウェブブラウザでは、インポートするモジュール名としてJavaScriptファイルの絶対URLまたは相対URLを指定することを説明できる

## 前提として使う既習知識

- 変数、関数、クラス、オブジェクトを宣言し、値やプロパティを読み書きできる
- 条件分岐、関数呼び出し、メソッド呼び出しを読める
- Promiseの結果を`.then`で扱える
- `async`関数、`await`式、`try...catch`を使った非同期処理を読める
- HTMLの`script`要素でJavaScriptファイルを読み込む基本的な形を読める

## 課題で要求しない知識（必要な場合のみ）

- CommonJSの`require`や`module.exports`は要求しない
- Node.jsでECMAScriptモジュールを実行するための設定やパッケージ管理の知識は要求しない
- バンドラー、トランスパイラ、開発サーバーの設定は要求しない
- モジュール解決、循環参照、ライブバインディングなど、本文で扱っていない詳細な仕様は要求しない
- DOM APIを使った画面操作や、`window`オブジェクトの詳細な知識は要求しない
- インポート属性で扱える`type`の網羅や、実行環境ごとの差分の詳細は要求しない

## 課題で扱う場合の注意

- 複数ファイルの課題にする場合は、各モジュールのファイル名、エクスポート名、インポート元の相対パスを明示する
- 実行を求める課題では、ブラウザで`type="module"`として読み込むことや、必要に応じてローカルサーバーを使うことを課題文で明示する
- `import`文や`export`文の構文エラーは実行環境で表示が異なるため、エラーメッセージの完全一致ではなく原因の説明を評価する
- Dynamic Importやインポート属性を実行させる課題では、Promise、`await`、インポート属性をサポートする実行環境を指定する
- インポート属性は実行環境によって対応状況が異なるため、構文読解を評価するのか実行結果を評価するのかを課題文で明示する
