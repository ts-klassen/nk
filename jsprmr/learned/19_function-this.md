# 19. 関数とthis

## このページで学ぶこと

- `this` は状況によって参照先が変わるキーワードであり、主にメソッドの中で利用することを学ぶ
- トップレベルの `this` は実行コンテキストが "Script" か "Module" かによって結果が異なることを学ぶ
- Arrow Function 以外の関数では、`this` が関数の定義時ではなく呼び出し時に決まることを学ぶ
- メソッド呼び出しにおけるベースオブジェクトと `this` の関係を学ぶ
- `this` を含むメソッドを変数へ代入した場合や、コールバック関数で `this` を参照した場合に起きる問題を学ぶ
- `call`、`apply`、`bind` や Arrow Function によって `this` の参照先を扱いやすくする方法を学ぶ
- Arrow Function は自身の `this` を持たず、外側の関数の `this` を参照することを学ぶ

## 課題で要求してよい知識

- `this` は読み取り専用のグローバル変数のようにどこにでも書けるが、参照先は条件によって変わることを説明できる
- `this` は基本的にメソッドの中で使い、トップレベルや通常の関数では使うべきではないことを説明できる
- 実行コンテキストが "Script" のトップレベルにある `this` はグローバルオブジェクトを参照することを説明できる
- 実行コンテキストが "Module" のトップレベルにある `this` は `undefined` になることを説明できる
- グローバルオブジェクトを参照したい場合は、トップレベルの `this` ではなく `globalThis` を使える
- オブジェクトのプロパティが関数である場合に、そのプロパティをメソッドと呼べる
- オブジェクトリテラルで、関数式をプロパティに代入する方法とメソッドの短縮記法でメソッドを定義できる
- `オブジェクト名.メソッド名()` の形でメソッドを呼び出せる
- Arrow Function 以外の関数やメソッドでは、`this` が呼び出し時に暗黙的に渡される値のように決まることを説明できる
- Arrow Function 以外の関数では、関数の定義だけを見ても `this` の参照先を決定できないことを説明できる
- メソッド呼び出しにおけるベースオブジェクトは、ドット演算子またはブラケット演算子のひとつ左にあるオブジェクトであることを説明できる
- `obj.method()` のようなメソッド呼び出しでは、`this` がベースオブジェクトである `obj` を参照することを説明できる
- `obj1.obj2.method()` や `obj1["obj2"]["method"]()` のような呼び出しでは、メソッドのひとつ左にある `obj2` がベースオブジェクトになることを説明できる
- `fn()` のようにベースオブジェクトがない関数呼び出しでは、strict mode における `this` が `undefined` になることを説明できる
- strict mode ではない状況で `this` が `undefined` になる場合、グローバルオブジェクトを参照するように変換されることを説明できる
- メソッド内で `this.プロパティ名` と書くことで、ベースオブジェクトのプロパティを参照できる
- `this` を含むメソッドを別の変数へ代入して通常の関数として呼び出すと、ベースオブジェクトがなくなり `this` の参照先が変わることを説明できる
- `this` に依存するメソッドは、基本的にメソッドとして呼び出すことで `this` の問題を避けられる
- `call` メソッドを使い、第一引数に `this` とする値、第二引数以降に関数の引数を渡して関数を呼び出せる
- `apply` メソッドを使い、第一引数に `this` とする値、第二引数に関数の引数の配列を渡して関数を呼び出せる
- `call` と `apply` の違いは、呼び出す関数へ引数を個別に渡すか配列として渡すかであることを説明できる
- `this` の値が不要な関数を `call` や `apply` で呼び出す場合、`this` の値として `null` を渡すのが一般的であることを説明できる
- `bind` メソッドを使い、`this` や一部の引数を束縛した新しい関数を作成できる
- `call`、`apply`、`bind` は、どうしても `this` を固定したい場合に利用するものだと説明できる
- メソッド内のコールバック関数を `function` キーワードで定義した場合、そのコールバック関数の `this` はコールバックの呼び出し方に依存することを説明できる
- `Array.prototype.map` に渡した通常のコールバック関数は、単なる関数として呼び出されるため、strict mode では `this` が `undefined` になることを説明できる
- メソッド呼び出し時の `this` を一時変数へ代入し、コールバック関数からその変数を参照することで `this` の問題を避けられる
- `Array.prototype.map` の第二引数に `this` とする値を渡すことで、通常のコールバック関数内の `this` を指定できる
- Arrow Function でコールバック関数を定義すると、外側の関数の `this` をそのまま参照できることを説明できる
- Arrow Function で定義された関数は自身の `this` を持たず、呼び出し時に暗黙的な `this` を受け取らないことを説明できる
- Arrow Function 内の `this` は、外側のスコープにあるもっとも近い関数の `this` を参照することを説明できる
- Arrow Function の外側に関数がない場合、Arrow Function 内の `this` はトップレベルの `this` と同じ値になることを説明できる
- Arrow Function の `this` は定義時の外側のスコープから決まり、呼び出し方では変化しないことを説明できる
- Arrow Function で定義したオブジェクトのメソッドは、呼び出し元のオブジェクトを `this` として受け取らないことを説明できる
- Arrow Function に対して `call`、`apply`、`bind` で `this` を指定しても、その Arrow Function 自身の `this` は変わらないことを説明できる
- Arrow Function が参照する外側の通常関数の `this` は、その外側の通常関数を `call` などで呼び出すことで変えられることを説明できる
- Script/Module、strict mode、関数定義の種類、呼び出し方をもとに、教材本文の表にある範囲で `this` の評価結果を予測できる

## 前提として使う既習知識

- strict mode と実行コンテキストの "Script" / "Module" の違いを説明できる
- 関数宣言、関数式、Arrow Function を定義して呼び出せる
- オブジェクトを作成し、ドット演算子やブラケット演算子でプロパティへアクセスできる
- オブジェクトのプロパティとして関数を持たせ、メソッドとして呼び出せる
- 配列の `map` メソッドにコールバック関数を渡して新しい配列を作成できる
- スコープチェーンと静的スコープの基本を説明できる

## 課題で要求しない知識（必要な場合のみ）

- コンストラクタ、クラス、インスタンス生成における `this` は要求しない
- `new`、`super`、継承、プロトタイプチェーンと `this` の関係は要求しない
- DOM API、イベントリスナー、ブラウザ固有 API における `this` の挙動は要求しない
- `import` / `export` などモジュール構文の詳細は要求しない
- `try...catch` による例外処理や、`TypeError` オブジェクトの詳細な扱いは要求しない

## 課題で扱う場合の注意

- トップレベルや通常関数の `this` を扱う課題では、実行コンテキストと strict mode の有無を課題文で明示する
- ブラウザの `window` や Node.js の `global` を直接前提にするのではなく、必要なら `globalThis` を使う前提にする
- `this` の束縛を扱う課題では、対象が Arrow Function か Arrow Function 以外の関数かを明示する
- コールバック関数内の `this` を扱う課題では、`function` キーワードを使うのか Arrow Function を使うのか、また `map` の第二引数を使ってよいのかを明示する
- `this` を含むメソッドを変数へ代入する問題では、メソッドとして呼んだ場合と通常の関数として呼んだ場合の違いを評価する
- 例外が発生するコードを扱う場合は、例外処理そのものではなく、`this` が `undefined` になる原因の説明を評価する
