# 14. 配列

## このページで学ぶこと

- 配列が順序を持つ要素を格納する可変長のオブジェクトであること
- 配列リテラル、インデックス、`length`、`at`、分割代入を使った配列要素の扱い
- 配列かどうかの判定、疎な配列、`undefined` の要素と未定義の要素の違い
- 要素の検索、追加、削除、結合、展開、範囲取得に使う代表的な Array メソッド
- 破壊的なメソッドと非破壊的なメソッドの違いと、配列をコピーして副作用を避ける考え方
- `forEach`、`map`、`filter`、`reduce`、`flat`、`flatMap`、`Object.groupBy` を使った配列の反復処理と加工
- Array-like オブジェクトと配列の違い、メソッドチェーンと高階関数の基本的な使い方

## 課題で要求してよい知識

- 配列が値を順番に格納し、要素の位置を 0 から始まるインデックスで表すことを説明できる
- JavaScript の配列が可変長であり、作成後に要素の追加や削除ができることを説明できる
- 配列リテラル `[]` を使って空の配列、複数要素の配列、配列の配列を作成できる
- `array[index]` を使って指定したインデックスの要素を読み取れる
- 2 次元配列に対して `array[0][0]` のように複数回インデックスアクセスして要素を読み取れる
- 配列の先頭要素のインデックスが `0` であることを前提に、アクセス結果を予測できる
- `length` プロパティが配列の要素数を返すことを説明できる
- `array.length - 1` を使って配列の最後の要素へアクセスできる
- 存在しないインデックスへアクセスすると例外ではなく `undefined` が返ることを説明できる
- 値を省略した配列リテラルによって疎な配列が作られることを説明できる
- 疎な配列と、すべてのインデックスに要素がある密な配列を区別できる
- `Array.prototype.at` を使って配列の指定位置の要素を取得できる
- `array.at(-1)` が末尾から 1 つ目の要素を返すことを説明できる
- `array[-1]` は末尾要素へのアクセスではなく `"-1"` プロパティへのアクセスになり、多くの場合 `undefined` になることを説明できる
- `Array.isArray` を使って値が配列かどうかを判定できる
- `typeof array` は `"object"` になるため、配列判定には使えないことを説明できる
- TypedArray は通常の `Array` とは別のオブジェクトであり、`Array.isArray` の結果が `false` になることを説明できる
- 配列の分割代入で、右辺の配列の対応するインデックスの要素を左辺の変数へ代入できる
- `undefined` という値を持つ要素と、疎な配列で要素自体が存在しない箇所は、読み取り結果だけでは区別できないことを説明できる
- `Object.hasOwn(array, index)` を使って、指定したインデックスに要素自体が存在するかを判定できる

- 配列から要素を検索するときに、欲しい結果がインデックス、要素そのもの、真偽値のどれかを判断できる
- `indexOf` が引数と `===` で一致する要素を先頭から検索し、見つかったインデックスまたは `-1` を返すことを説明できる
- `lastIndexOf` が末尾から検索し、見つかったインデックスまたは `-1` を返すことを説明できる
- `indexOf` や `lastIndexOf` で得たインデックスを使って、該当する配列要素へアクセスできる
- オブジェクトは同じプロパティを持っていても参照が異なると `indexOf` では一致しないことを説明できる
- `findIndex` にコールバック関数を渡し、条件に一致する最初の要素のインデックスを取得できる
- `findLastIndex` が条件に一致する最後の要素のインデックスを返すことを説明できる
- `find` にコールバック関数を渡し、条件に一致する最初の要素そのものを取得できる
- `find` が該当要素を見つけられない場合に `undefined` を返すことを説明できる
- `findLast` が条件に一致する最後の要素そのものを返すことを説明できる
- `slice(start, end)` が開始位置から終了位置の直前までを取り出した新しい配列を返すことを説明できる
- `slice` の第二引数を省略すると、開始位置から末尾までを取り出すことを説明できる
- `slice` に負の値を渡すと末尾から数えた位置として扱われることを説明できる
- `slice` で開始位置と終了位置が同じ場合や開始位置が終了位置より後の場合、空配列が返ることを説明できる
- 要素が含まれているかだけを知りたい場合に `includes` を使って真偽値を取得できる
- `includes` は `indexOf` と同様に、値が同じでも参照が異なるオブジェクトの検索には向かないことを説明できる
- 条件に一致する要素が存在するかを知りたい場合に `some` を使って真偽値を取得できる

- `push` を使って配列の末尾へ要素を追加できる
- `pop` を使って配列の末尾の要素を削除し、その要素を取得できる
- `unshift` を使って配列の先頭へ要素を追加できる
- `shift` を使って配列の先頭の要素を削除し、その要素を取得できる
- `concat` を使って配列と配列、または配列と任意の値を結合した新しい配列を作成できる
- Spread 構文 `...array` を配列リテラルの中で使い、既存の配列の要素を任意の位置へ展開できる
- `splice(index, deleteCount)` を使って任意のインデックスから指定個数の要素を削除できる
- `splice(index, deleteCount, ...items)` を使って削除と同時に要素を追加できることを説明できる
- `splice` で削除したあとの要素は詰められ、削除によって疎な配列にはならないことを説明できる
- `array.length = 0` によって配列の要素をすべて削除できることを説明できる
- `length` へ要素数を代入すると、その要素数に配列が切り詰められることを説明できる
- 変数へ `[]` を再代入すると、元の配列を変更するのではなく新しい空配列を参照することを説明できる
- `const` で宣言した配列変数には新しい配列を再代入できないことを説明できる

- 破壊的なメソッドが配列オブジェクトそのものを変更するメソッドであることを説明できる
- 非破壊的なメソッドが元の配列を変更せず、新しい配列を返すメソッドであることを説明できる
- `push`、`pop`、`splice`、`shift`、`unshift` が破壊的なメソッドであることを判断できる
- `concat`、`slice`、`map`、`filter`、`flat`、`flatMap` が新しい配列を返す非破壊的なメソッドであることを判断できる
- `sort`、`reverse`、`copyWithin`、`fill` は破壊的なメソッドとして紹介されていることを説明できる
- メソッド名や返り値だけでは破壊的か非破壊的かを判別できない場合があることを説明できる
- 破壊的なメソッドを関数内で使うと、引数として受け取った配列にも副作用を与えることを説明できる
- `slice()` や `concat()` を引数なしで呼び出すと配列のコピーを作れることを説明できる
- コピーした配列に破壊的な変更を加えることで、元の配列への副作用を避けられることを説明できる
- `toSpliced` を使って、`splice` と同じ引数で非破壊的に変更した新しい配列を作成できる
- `toReversed`、`toSorted` がそれぞれ `reverse`、`sort` の非破壊的な対応メソッドであることを説明できる
- `with(index, value)` を使って、指定インデックスの要素を非破壊的に変更した新しい配列を作成できる
- `array[index] = value` による代入が元の配列を変更する破壊的な処理であることを説明できる
- まず非破壊的な方法で書けるか検討し、必要な場合に破壊的な方法を使うという判断ができる

- `forEach` が配列の要素を先頭から順番にコールバック関数へ渡して反復処理することを説明できる
- `forEach`、`map`、`filter` のコールバック関数に `要素, インデックス, 配列` が渡されることを説明できる
- `map` が各要素をコールバック関数で加工し、その返り値を集めた新しい配列を返すことを説明できる
- `filter` がコールバック関数で `true` を返した要素だけを集めた新しい配列を返すことを説明できる
- `reduce` が累積値と各要素をコールバック関数に渡し、配列から 1 つの値を作ることを説明できる
- `reduce` のコールバック関数に `累積値, 要素, インデックス, 配列` が渡されることを説明できる
- `reduce` の第二引数で累積値の初期値を指定できる
- `reduce` を使って数値の合計やオブジェクトへの集約のように、配列から配列以外の値を作成できる
- `reduce` は便利だが複雑になりやすいため、処理の意図が伝わるように関数化するなどの工夫が必要であることを説明できる
- `flat` を使って多次元配列を指定した深さまでフラットな配列に変換できる
- `flat()` の引数なしは 1 段階のフラット化であり、`flat(Infinity)` ですべての階層をフラット化できることを説明できる
- `flat` はこれ以上フラット化できない配列に対しても新しい配列を返すことを説明できる
- `flatMap` が各要素にコールバック関数を適用し、その結果を 1 段階フラット化した新しい配列を返すことを説明できる
- `flatMap` は `map` してから `flat(1)` するのと同じ結果になることを説明できる
- `Object.groupBy(array, callback)` を使って、コールバック関数が返したキーごとに配列要素をグループ分けしたオブジェクトを作成できる

- Array-like オブジェクトが、インデックスアクセスと `length` を持つが、配列のインスタンスではないオブジェクトであることを説明できる
- `arguments` オブジェクトが Array-like オブジェクトの例であり、配列メソッドを持たない場合があることを説明できる
- `Array.isArray` を使って Array-like オブジェクトと配列を区別できる
- `Array.from` を使って Array-like オブジェクトを配列に変換できる
- メソッドチェーンが、メソッドの返り値に対して続けてメソッドを呼び出すパターンであることを説明できる
- `concat`、`filter`、`map` など配列を返すメソッドをつなげて、途中の一時変数を省略した処理を書ける
- 長すぎるメソッドチェーンは読みづらくなるため、適度な単位で使う必要があることを説明できる
- コールバック関数を受け取る配列メソッドが高階関数として扱われることを説明できる

## 前提として使う既習知識

- 変数宣言、再代入、`const` と `let` の違いを使える
- 数値、文字列、真偽値、`undefined`、オブジェクト、関数を扱える
- オブジェクトのプロパティを読み書きできる
- 関数宣言、アロー関数、コールバック関数を使える
- `if` 文、比較演算子、厳密等価演算子、論理演算子を使える
- `for...of` や既習の反復処理の考え方を使える
- `console.log` で値を確認できる

## 課題で要求しない知識（必要な場合のみ）

- 教材で詳しく説明していない Array メソッドの詳細な挙動は要求しない
- `sort`、`reverse`、`copyWithin`、`fill` の具体的な使い方は、破壊的メソッドとしての判別以上には要求しない
- TypedArray のバイナリデータ処理、WebGL、バッファ操作の知識は要求しない
- Array-like オブジェクトとして DOM API の戻り値を扱う知識は要求しない
- イテレータや iterable オブジェクトの詳細な仕様理解は要求しない
- ガベージコレクションの詳細な仕組みは要求しない
- MDN や仕様書を参照しないと分からない Array メソッドの網羅的な知識は要求しない

## 課題で扱う場合の注意

- 破壊的な操作を許可するか、非破壊的な実装を求めるかを課題文で明示する
- メソッドの返り値を評価するのか、元の配列が変更されたかを評価するのかを明示する
- `slice`、`splice`、`toSpliced` は名前が似ているため、どのメソッドを使う課題なのかを明示する
- `at`、`findLast`、`findLastIndex`、`toSpliced`、`toReversed`、`toSorted`、`with`、`Object.groupBy` は比較的新しい ECMAScript の機能なので、実行環境が対応している前提を課題文に含める
- 疎な配列や `Object.hasOwn` を扱う課題では、値が `undefined` であることと要素自体が存在しないことのどちらを判定するのかを明示する
- `reduce` を要求する課題では、初期値の指定と累積値の型を明示する
- 複数の Array メソッドで自然に解ける課題では、特定のメソッドを練習させるのか、結果だけを評価するのかを明示する
