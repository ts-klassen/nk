# 30. 第一部: おわりに

## このページで学ぶこと

- 第一部では、ECMAScript という仕様の範囲で JavaScript の基本文法や使い方を扱ったことを確認する
- 第二部では、第一部の基本文法を応用したアプリケーション実装に加えて、Node.js やブラウザ固有の API、ライブラリの利用方法も扱うことを確認する
- この書籍では ECMAScript のすべての文法やビルトインオブジェクトを扱っているわけではなく、必要になったときに調べて覚える対象があることを確認する
- JavaScript、Node.js、ブラウザ API、ライブラリやツールについて、公式のリファレンスやリポジトリを参照して調べる重要性を扱う

## 課題で要求してよい知識

- 第一部で扱った範囲が、ECMAScript の基本文法やビルトインオブジェクトを中心とした内容であることを説明できる
- 第二部で扱う範囲には、Node.js やブラウザ固有の API、ライブラリの利用方法が含まれることを説明できる
- 書籍で紹介されていない ECMAScript の文法やビルトインオブジェクトが存在し、必要になったときに調べて使い方を覚える対象であることを説明できる
- `Proxy` や `Reflect` は、オブジェクトの基本的な操作に対して独自の動作を定義できるビルトインオブジェクトの例であると説明できる
- `Object.defineProperty` は、オブジェクトの記述子を変更して、プロパティを変更できなくするようなメタ的な動作を設定できるメソッドの例であると説明できる
- `Proxy`、`Reflect`、`Object.defineProperty` のような API は、アプリケーションよりもライブラリ作成で利用されることが多く、この書籍では詳しい使い方を省略していることを説明できる
- JavaScript の多くの API や DOM API を調べるときに、MDN Web Docs をリファレンスとして参照できる
- Node.js 固有の API を調べるときに、Node.js Documentation を公式リファレンスとして参照できる
- ライブラリやツールの使い方を調べるときに、その公式サイトやリポジトリを参照できる
- ECMAScript、JavaScript、ライブラリは変化し続けるため、Deprecated になっている可能性も含めて、元となる公式情報を確認する重要性を説明できる
- 調べ方に唯一の正解はないが、調べたいときに適切なリファレンスを探して参照することが重要であると説明できる

## 前提として使う既習知識

- ECMAScript が JavaScript の仕様であることを説明できる
- JavaScript の基本文法、ビルトインオブジェクト、オブジェクトのプロパティ操作について、第一部で扱った範囲を使える
- JavaScript の機能や仕様が更新されるため、利用する環境や仕様のバージョンによって使える機能が異なる場合があることを説明できる

## 課題で要求しない知識（必要な場合のみ）

- `Proxy` や `Reflect` を使った具体的な実装は要求しない
- `Object.defineProperty` の詳細な記述子オプションや、それらを使った実装は要求しない
- DOM API、Node.js API、ライブラリ、ツールの具体的な使い方は、このページだけを根拠には要求しない
- MDN Web Docs や Node.js Documentation に載っている未学習 API の詳細な仕様理解は要求しない

## 課題で扱う場合の注意

- このページは第一部のまとめと調べ方の紹介が中心であるため、課題では未紹介 API の実装能力ではなく、参照先の選び方や学習範囲の区別を評価する
- 公式リファレンスの内容を確認させる課題にする場合は、外部サイトの最新情報が変わる可能性を考慮し、特定の細かな文言や表示位置への依存を避ける
- `Proxy`、`Reflect`、`Object.defineProperty` は例として紹介されているだけなので、課題では名前と大まかな用途の説明に留める
