# 23. Map/Set

## このページで学ぶこと

- キーと値の組み合わせを扱う `Map` と、重複しない値の集合を扱う `Set` の基本的な使い方
- `Map` と `Set` の作成、初期化、追加、取得、存在確認、削除、全削除、要素数の確認
- `Map` と `Set` の反復処理と、配列への変換
- マップとしての `Object` と `Map` の違い、`WeakMap` と `WeakSet` の特徴
- `Map.groupBy` 静的メソッド、`Set` の集合演算メソッド、`Map` のキー比較における `NaN` の扱い

## 課題で要求してよい知識

- `new Map()` で空のマップを作成し、`size` プロパティで要素数を確認できる
- エントリーの配列を `Map` コンストラクタに渡して、初期値を持つマップを作成できる
- `Map.prototype.set` でキーと値の組み合わせを追加し、同じキーに対する追加が値を上書きすることを説明できる
- `Map.prototype.get` でキーにひもづいた値を取り出し、`Map.prototype.has` でキーの存在を確認できる
- `Map.prototype.delete` で指定したキーと値を削除し、`Map.prototype.clear` ですべての要素を削除できる
- `Map.prototype.forEach` のコールバックには値、キー、マップの順に引数が渡され、挿入順に反復処理されることを説明できる
- `Map.prototype.keys`、`Map.prototype.values`、`Map.prototype.entries` の返り値を `for...of` 文や `Array.from` で扱える
- `Map.prototype.entries` や `for...of` による `Map` 自体の反復処理で、`[キー, 値]` のエントリーを挿入順に取り出せる
- Spread構文で `Map` をエントリーの配列へ展開し、`keys` や `values` の返り値を配列へ展開できる
- `Map.groupBy` にiterableオブジェクトとグループ分け条件を返すコールバック関数を渡し、グループごとの配列を値に持つ `Map` を作成できる
- マップとして `Object` を使う場合は継承したプロパティとの衝突や、キーに使える値が文字列または `Symbol` に限られる問題があることを説明できる
- `Map` はプロパティとは異なる仕組みでデータを格納するため、プロトタイプのメソッドやプロパティとキーが衝突しないことを説明できる
- `Map` ではオブジェクトをキーにでき、見た目が同じでも参照が異なるオブジェクトは別のキーとして扱われることを説明できる
- `Object` のリテラル表現、JSONへの変換しやすさ、既存APIとの相性が、簡易なマップ用途で利点になることを説明できる
- `WeakMap` はキーを弱い参照で持つマップであり、キーとして使えるのは参照型のオブジェクトと `Symbol` であることを説明できる
- `WeakMap` はiterableではなく、`keys` メソッドや `size` プロパティを持たないことを説明できる
- `WeakMap` を使って、クラスインスタンスにひもづく外部からアクセスできない値や、一時的な計算結果を保存する用途を説明できる
- `Map` のキー比較では基本的に `===` と同じように判定されるが、`NaN` 同士は等価なキーとして扱われることを説明できる
- `new Set()` で空のセットを作成し、`size` プロパティで要素数を確認できる
- iterableオブジェクトを `Set` コンストラクタに渡して、初期値を持つセットを作成できる
- `Set` は重複する同じ値を1つだけ保持し、インデックスによるアクセスはできないことを説明できる
- `Set.prototype.add` で値を追加し、すでに存在する値を追加しても重複して格納されないことを説明できる
- `Set.prototype.has` で値の存在を確認し、`Set.prototype.delete` で指定した値を削除し、`Set.prototype.clear` ですべての値を削除できる
- `Set.prototype.forEach` と `for...of` 文で、セットの値を挿入順に反復処理できる
- `Set.prototype.keys` は `values` のエイリアスであり、`Set.prototype.entries` は `[値, 値]` のエントリーを返すことを説明できる
- Spread構文で `Set` を配列へ展開できる
- `Set.prototype.union`、`intersection`、`difference`、`symmetricDifference` を使って、新しい `Set` として和集合、積集合、差集合、対称差集合を作成できる
- `Set.prototype.isSubsetOf`、`isSupersetOf`、`isDisjointFrom` を使って、集合の関係を真偽値として判定できる
- `WeakSet` は値を弱い参照で持つセットであり、値として使えるのは参照型のオブジェクトと `Symbol` であることを説明できる
- `WeakSet` はiterableではなく、値の追加、削除、存在確認以外の操作や `Set` の集合演算メソッドを利用できないことを説明できる

## 前提として使う既習知識

- 変数宣言、関数定義、アロー関数、条件分岐を使える
- オブジェクトと配列を作成し、プロパティや要素へアクセスできる
- クラス、インスタンス、`this` の基本的な意味を説明できる
- `for...of` 文、配列の `forEach`、`Array.from`、Spread構文、分割代入を使える
- `Symbol`、`NaN`、`===` 演算子の基本的な意味を説明できる

## 課題で要求しない知識（必要な場合のみ）

- Iteratorプロトコルやiterableプロトコルを自分で実装する知識は要求しない
- ガベージコレクションの実行タイミングやメモリ解放を観測する知識は要求しない
- DOM API、フォームイベント、`fetch` によるHTTP通信、HTML要素の計測処理は要求しない
- `Object.create(null)` の詳細な使い方や、プロトタイプチェーンの詳細な実装知識は要求しない
- `Map` や `Set` の内部実装、計算量、仕様アルゴリズムの詳細は要求しない

## 課題で扱う場合の注意

- `Map.groupBy` はES2024、`Set` の集合演算メソッドはES2025の機能であるため、実行環境で使えるかを課題文やテスト環境で明示する
- `WeakMap` と `WeakSet` は列挙できないため、課題では `keys`、`values`、`entries`、`for...of`、`size` を使う解法を要求しない
- `WeakMap` のキーや `WeakSet` の値が自動的に解放されることは、実行タイミングに依存するためテストの期待結果にしない
- `Set` はインデックスによるアクセスはできないが、反復処理では挿入順に値が取り出される点を区別して扱う
- マップ用途で `Object` と `Map` のどちらを使うかを問う場合は、キーの種類、プロパティ衝突、JSON変換、既存APIとの相性のどれを判断基準にするかを明示する
