# 27. Math

## このページで学ぶこと

- 数学的な定数や関数を提供するビルトインオブジェクトである `Math` の基本的な使い方
- `Math` の定数や関数が、インスタンスではなく静的プロパティや静的メソッドとして提供されること
- `Math.random` による疑似乱数の生成
- `Math.max` と `Math.min` による数値の最大値・最小値の取得
- `Math.floor`、`Math.ceil`、`Math.round`、`Math.trunc` による数値の整数化

## 課題で要求してよい知識

- `Math` が数学的な定数や関数を提供するビルトインオブジェクトであることを説明できる
- `Math` はコンストラクタではなく、`new Math()` のようにインスタンス化して使うものではないことを説明できる
- `Math.PI` のような静的プロパティや、`Math.sin(...)` のような静的メソッドとして `Math` の機能を利用できる
- `Math.random()` が `0` 以上 `1` 未満の疑似ランダムな浮動小数点数を返すことを説明できる
- `Math.random() * (max - min) + min` の形で、`min` 以上 `max` 未満の範囲の乱数を生成できる
- `Math.max(...)` が渡された複数の数値の最大値を返すことを説明し、結果を予測できる
- `Math.min(...)` が渡された複数の数値の最小値を返すことを説明し、結果を予測できる
- 数値の配列に spread 構文を使い、`Math.max(...numbers)` や `Math.min(...numbers)` で最大値・最小値を取得できる
- `Math.floor(number)` が、引数以下で最大の整数を返すことを説明し、正の数と負の数の結果を予測できる
- `Math.ceil(number)` が、引数以上で最小の整数を返すことを説明し、正の数と負の数の結果を予測できる
- `Math.round(number)` が、小数部分に応じて四捨五入した整数を返すことを説明し、基本的な結果を予測できる
- `Math.trunc(number)` が、小数部分を単純に切り落とした整数を返すことを説明し、正の数と負の数の結果を予測できる
- 正の数では `Math.trunc` と `Math.floor` の結果が同じになり、負の数では `Math.trunc` と `Math.ceil` の結果が同じになることを説明できる

## 前提として使う既習知識

- 変数宣言、関数定義、引数、戻り値を使える
- 数値リテラルと算術演算子を使って計算できる
- `for` 文で同じ処理を繰り返せる
- 配列を作成し、spread 構文で配列の要素を引数として展開できる
- `console.log` で式の評価結果を確認できる

## 課題で要求しない知識（必要な場合のみ）

- このページで紹介されていない `Math` の定数やメソッドの網羅的な知識は要求しない
- `Math.random` の乱数の品質、シード、暗号用途での安全性に関する知識は要求しない
- 浮動小数点数の丸め誤差や、`Math.round` の境界値に関する仕様の詳細は要求しない
- 三角関数全般や角度・ラジアン変換の数学的な知識は、`Math.PI` と `Math.sin` の例を読める範囲を超えて要求しない

## 課題で扱う場合の注意

- `Math.random` を使う課題では、戻り値の完全一致ではなく、生成される値の範囲や式の構造を評価する
- 最大値・最小値を配列から求める課題では、spread 構文を使う前提か、他の方法も許可するかを課題文で明示する
- 整数化メソッドを扱う課題では、`floor`、`ceil`、`round`、`trunc` のどれを使うべきか、または違いを説明させるのかを課題文で明示する
- 負の数を含む丸めの課題では、期待する結果を具体例で確認できるようにする
