# 03. 変数と宣言

## このページで学ぶこと

- 変数は、文字列や数値などのデータに名前をつけて繰り返し利用するための仕組みである
- JavaScript には変数宣言のキーワードとして `const`、`let`、`var` がある
- `const`、`let`、`var` で宣言した変数の再代入、初期値、再定義の違いを扱う
- 変数名として利用できる識別子の基本的なルールを扱う
- 新しく書くコードでは、まず `const` を検討し、再代入が必要な場合は `let` を使い、`var` は避けることが推奨される

## 課題で要求してよい知識

- 変数が、データに名前をつけて再利用するための仕組みであることを説明できる
- `const 変数名 = 初期値;` の形で、再代入できない変数を宣言できる
- `const`、`let`、`var` のいずれでも、カンマで区切って複数の変数を同時に宣言できることを説明できる
- `const` で宣言した変数へ後から値を代入しようとすると `TypeError` になることを予測できる
- 再代入が不要な変数には `const` を選ぶ判断ができる
- `let` で、初期値を持つ変数と初期値を持たない変数を宣言できる
- 初期値を指定しない `let` 変数は `undefined` で初期化されることを説明できる
- `let` で宣言した変数には代入演算子 `=` で値を代入でき、何度でも再代入できることを説明できる
- `var` は `let` と同じように初期値なしの宣言や再代入ができることを説明できる
- `let` や `const` では同じ名前の変数を再定義しようとすると `SyntaxError` になることを予測できる
- `var` では同じ名前の変数を再定義でき、意図せず値を上書きする問題があることを説明できる
- `var` には再定義や巻き上げなどの問題があり、ほとんどの場合は `const` または `let` に置き換えられることを説明できる
- `const` と `let` が、`var` の既存の挙動を変えずに問題を避けるため ECMAScript 2015 で追加されたことを説明できる
- 変数名には半角アルファベット、`_`、`$`、数字を組み合わせて使えることを説明できる
- 変数名に数字を含めることはできるが、数字から始めることはできないと判断できる
- `let` や `if` のような予約語は変数名に使えないと判断できる
- ひらがなや一部の漢字も変数名に利用できるが、全角文字の混在は推奨されないことを説明できる
- `const` は「再代入できない変数」を宣言するものであり、命名規則や代入できる値を制限するものではないことを説明できる

## 前提として使う既習知識

- JavaScript のコード例を読み、文末のセミコロンを含む基本的な文として扱える
- 文字列や数値のリテラルを、変数に代入する値として扱える
- `//` から始まるコメントを、コードの説明として読み分けられる

## 課題で要求しない知識（必要な場合のみ）

- `var` の巻き上げやスコープの詳細な挙動は要求しない
- ループなどの反復処理の構文は要求しない
- オブジェクトを `const` で宣言した場合のプロパティ変更など、オブジェクトに関する詳細は要求しない
- 予約語の網羅的な一覧や Unicode 識別子の詳細仕様は要求しない
- DOM API、ブラウザ操作、Node.js 固有の知識は要求しない

## 課題で扱う場合の注意

- 新しいコードを書かせる課題では、原則として `const` を優先し、再代入が必要な場合だけ `let` を使わせる
- `var` は、新規コードで使わせるよりも、`let` や `const` との違いや問題点を読解・説明させる用途に限定する
- エラーを扱う課題では、エラーメッセージの完全一致ではなく、`TypeError` や `SyntaxError` になる原因を評価する
- カンマ区切りの複数宣言を扱う場合は、個別に宣言した場合と同じ意味になることを確認する程度に留める
