# 02. コメント

## このページで学ぶこと

- コメントはプログラムとして評価されず、ソースコードに説明を書くために使うことを学ぶ
- JavaScript のコメントには、一行コメントと複数行コメントがあることを学ぶ
- 複数行コメントはネストできないことを学ぶ
- HTML-like コメントは後方互換性のためだけに存在する仕様であることを学ぶ

## 課題で要求してよい知識

- コメントとして書かれた内容が JavaScript のコードとして評価されないことを説明できる
- `//` 以降から行末までが一行コメントとして扱われることを説明できる
- 一行コメントを使って、1 行の説明やコードとして評価されない行を書ける
- `/*` と `*/` で囲まれた範囲が複数行コメントとして扱われることを説明できる
- 複数行コメントを使って、複数行にわたる説明をコードとして評価されない形で書ける
- 複数行コメントの中に複数行コメントをネストして書けないことを説明できる
- ネストされた複数行コメントが構文エラーになることを判断できる
- HTML-like コメントが通常のコメント記法ではなく、後方互換性のために存在することを説明できる
- `<!--` と `-->` が HTML-like コメントとして一行コメントのように扱われる例を読める

## 前提として使う既習知識

- JavaScript のソースコードは、実行時にコードとして評価される部分と評価されない部分に分けられることを前提にできる

## 課題で要求しない知識（必要な場合のみ）

- HTML-like コメントを新しく書くことは要求しない
- `<script>` タグやブラウザの歴史的な互換性の詳細は要求しない
- DOM API やブラウザ上での画面操作は要求しない

## 課題で扱う場合の注意

- コメントの内容そのものではなく、どの範囲がコメントとして評価されないかを評価対象にする
- HTML-like コメントは後方互換性の話として扱い、通常のコメント記法として利用させる課題にはしない
