# 12. オブジェクト

## このページで学ぶこと

- オブジェクトはプロパティの集合であり、プロパティはキーと値の組み合わせであること
- オブジェクトリテラルによるオブジェクトの作成、プロパティの定義、参照、追加、削除
- ドット記法、ブラケット記法、分割代入、Computed property names の使い分け
- オブジェクトがミュータブルであり、`const` で宣言してもプロパティは変更できること
- 存在しないプロパティへのアクセス、プロパティの存在確認、Optional chaining 演算子
- `toString` メソッド、プロパティ名の文字列化、`Symbol` キーの例外
- `Object` の静的メソッドによる列挙、マージ、複製

## 課題で要求してよい知識

- オブジェクトをキーと値の組み合わせであるプロパティの集合として説明できる
- プロパティキーには文字列または `Symbol` を使え、プロパティ値には任意の値を指定できることを説明できる
- オブジェクトリテラル `{}` を使って、空のオブジェクトや初期プロパティを持つオブジェクトを作成できる
- オブジェクトリテラルでプロパティ名と値を `:` で区切り、複数のプロパティを `,` で区切って定義できる
- 識別子として使えるプロパティ名はクォートを省略でき、識別子として使えない名前はクォートで囲む必要があることを判断できる
- プロパティ名と値に指定する変数名が同じ場合に、プロパティの省略記法 `{ name }` を使える
- オブジェクトリテラルは `Object` のインスタンスオブジェクトを作成する構文であり、通常は `new Object()` よりもオブジェクトリテラルを使うことを説明できる
- ドット記法 `object.property` とブラケット記法 `object["property"]` でプロパティ値を参照できる
- ドット記法は識別子として使えるプロパティ名にだけ使えることを判断できる
- ブラケット記法では式や変数の評価結果をプロパティ名として使えることを説明できる
- ブラケット記法で指定したプロパティ名は、`Symbol` を除いて文字列へ暗黙的に変換されることを説明できる
- 同じオブジェクトの複数プロパティを変数へ取り出すときに、オブジェクトの分割代入を使える
- 存在しないプロパティへ値を代入すると、そのプロパティが追加されることを説明できる
- ドット記法またはブラケット記法を使って、作成済みのオブジェクトへプロパティを追加できる
- オブジェクトリテラル内の Computed property names で、式の評価結果をプロパティ名にできる
- 作成後のオブジェクトへプロパティを追加すると、オブジェクトが持つプロパティがわかりにくくなるため、できるだけ作成時に定義する方針を説明できる
- `delete` 演算子を使って、オブジェクトからプロパティを削除できる
- `const` は変数への再代入を防ぐが、変数が参照するオブジェクトのプロパティ変更は防がないことを説明できる
- strict mode と `Object.freeze` を組み合わせると、凍結したオブジェクトへのプロパティ追加や変更で例外が発生することを説明できる
- 存在しないプロパティへのアクセスは例外ではなく `undefined` を返すことを説明できる
- `undefined` や `null` に対してさらにプロパティアクセスすると例外が発生することを説明できる
- `undefined` との比較では、存在しないプロパティと値が `undefined` のプロパティを区別できないことを説明できる
- プロパティの存在確認に `in` 演算子または `Object.hasOwn` 静的メソッドを使える
- `Object.hasOwn` は対象オブジェクトとプロパティ名を引数に取り、指定したプロパティを持つ場合に `true` を返すことを説明できる
- `Object.prototype.hasOwnProperty` は `Object.hasOwn` と似た存在確認をインスタンスから行うメソッドだが、利用できるなら `Object.hasOwn` を選ぶことを説明できる
- Optional chaining 演算子 `?.` を使って、ネストしたプロパティへのアクセス時に `null` または `undefined` で評価を止め、`undefined` を返せる
- Optional chaining 演算子をブラケット記法と組み合わせた `object?.[key]` の形で使える
- オブジェクトの `toString` メソッドがオブジェクト自身を文字列化し、`String` コンストラクタ関数もオブジェクトの `toString` メソッドを呼び出すことを説明できる
- 独自の `toString` メソッドを定義したオブジェクトでは、その返り値が `String(object)` の結果になることを説明できる
- オブジェクトをプロパティ名として使うと文字列化され、同じ `"[object Object]"` というプロパティ名として衝突し得ることを説明できる
- `Symbol` は文字列化されずにプロパティ名として扱える例外であることを説明できる
- インスタンスメソッドと静的メソッドの呼び出し元の違いを、`obj.toString()` と `Object.hasOwn(...)` のような例で区別できる
- `Object.keys` はプロパティ名の配列、`Object.values` は値の配列、`Object.entries` は `[キー, 値]` の配列を返すことを説明できる
- `Object.keys` などで得た配列を使って、オブジェクトのプロパティを列挙する処理を読解できる
- `Object.assign(target, ...sources)` は `sources` のプロパティを `target` へコピーし、返り値として `target` を返すことを説明できる
- `Object.assign({}, objectA, objectB)` のように空のオブジェクトを `target` にして、既存オブジェクトを変更せずにマージしたオブジェクトを作成できる
- `Object.assign` の第一引数に既存オブジェクトを指定すると、そのオブジェクト自体が変更されることを説明できる
- `Object.assign` で同じプロパティ名が重複した場合、後ろのオブジェクトのプロパティで上書きされることを説明できる
- オブジェクトの spread 構文 `{ ...objectA, ...objectB }` で、新しいオブジェクトとしてマージできる
- オブジェクトの spread 構文でプロパティ名が重複した場合、後ろに書いたプロパティが優先されることを説明できる
- `Object.assign({}, obj)` によってオブジェクトを浅く複製できる
- shallow copy は直下のプロパティだけをコピーし、ネストしたオブジェクトまでは複製しないことを説明できる
- deep copy はプロパティ値のオブジェクトまで再帰的に複製するコピーであることを、shallow copy と区別できる

## 前提として使う既習知識

- `const` による変数宣言、再代入不可、識別子として使える名前のルールを使える
- 文字列、数値、真偽値、`undefined`、`null`、`Symbol` などの基本的な値を扱える
- 比較演算子、論理演算子、代入演算子、`typeof` 演算子を使える
- `if` 文で条件分岐を書ける
- 関数を定義し、引数と返り値を扱える
- 配列を値として扱い、基本的な反復処理を読解できる
- strict mode の有無がエラー発生の挙動に影響する場合があることを説明できる

## 課題で要求しない知識（必要な場合のみ）

- プロトタイプオブジェクトの仕組みや、`in` 演算子と `Object.hasOwn` の厳密な違いは要求しない
- `Object.prototype.hasOwnProperty` の欠点の詳細は要求しない
- `Map`/`Set` を使ったオブジェクトキーの扱いは要求しない
- npm、外部ライブラリ、パッケージ管理の知識は要求しない
- DOM API、ブラウザ API、Node.js 固有 API は要求しない
- プロパティ属性、列挙可能性、getter/setter、`this` の詳細は要求しない
- 循環参照や特殊なオブジェクトに対応する汎用的な deep copy の実装は要求しない
- `Array.prototype.filter` や `Array.prototype.forEach` など配列メソッドの詳細な使い分けは要求しない

## 課題で扱う場合の注意

- `Object.hasOwn` は ES2022、Optional chaining 演算子は ES2020、オブジェクトの spread 構文は ES2018 の機能であるため、課題で使う場合は実行環境または代替手段を明示する
- `Object.freeze` による変更不可をエラーとして確認させる場合は、strict mode を使う前提を明示する
- プロパティの存在そのものを判定したいのか、取得した値が `undefined` かを判定したいのかを課題文で区別する
- `Object.assign` は第一引数を変更するため、既存オブジェクトを変更してよいかどうかを課題文で明示する
- マージで同名プロパティがある場合は、後ろの値で上書きされることを採点条件に含める
- shallow copy と deep copy を扱う課題では、ネストしたオブジェクトが同じ参照のままかどうかを評価対象として明示する
