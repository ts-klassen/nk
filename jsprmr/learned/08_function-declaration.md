# 08. 関数と宣言

## このページで学ぶこと

- 関数を一連の手続きとして定義し、呼び出して再利用する方法
- 関数宣言、関数式、Arrow Function による関数の定義方法
- 仮引数、引数、返り値、`return` 文、デフォルト引数の扱い
- 可変長引数を扱う Rest parameters、Spread 構文、`arguments` 変数
- 関数を値として扱う性質、コールバック関数、高階関数、メソッドの考え方

## 課題で要求してよい知識

- 関数が、文の集まりを 1 つの処理としてまとめ、呼び出しによって同じ処理を実行できることを説明できる
- `function 関数名(仮引数) { ... }` の形で関数宣言を書ける
- 関数名、仮引数、関数の中身、返り値という関数の構成要素を区別できる
- 複数の仮引数や引数を `,` で区切って書ける
- `関数名()` や `関数名(引数1, 引数2)` の形で関数を呼び出せる
- 関数の呼び出し結果として、`return` 文で指定した値が呼び出し元へ返ることを説明できる
- `return` 文が実行されると、その関数内のそれ以降の処理は実行されないことを説明できる
- `return` 文の返り値を省略した場合や、`return` 文そのものを省略した場合に `undefined` が返ることを説明できる
- 仮引数より呼び出し時の引数が少ない場合、対応する引数がない仮引数には `undefined` が入ることを説明できる
- 仮引数より呼び出し時の引数が多い場合、余分な引数は無視されることを説明できる
- `function 関数名(仮引数 = デフォルト値) { ... }` の形でデフォルト引数を指定できる
- 引数が渡されなかった場合にデフォルト引数の値が使われ、`||` による指定とは異なり空文字列などの値を渡した場合はその値が使われることを説明できる
- `||` を使ったデフォルト値指定では falsy な値にもデフォルト値が使われるため、意図しない結果になる場合があることを説明できる
- `??` を使うと `null` または `undefined` の場合だけデフォルト値を使えることを、本文の範囲で説明できる
- 任意の個数の引数を受け取る関数を可変長引数の関数として説明できる
- `...仮引数名` の Rest parameters で、渡された引数を配列として受け取れる
- 通常の仮引数と Rest parameters を組み合わせる場合、Rest parameters は末尾の仮引数として定義する必要があることを説明できる
- 関数呼び出しで配列の前に `...` をつける Spread 構文により、配列の要素を引数として展開できる
- Rest parameters が引数を配列にまとめる構文で、Spread 構文が配列を引数へ展開する構文であることを区別できる
- `arguments` が関数の中で参照でき、実際に渡された引数をすべて含む Array-like オブジェクトであることを説明できる
- `arguments` はインデックスで要素へアクセスできるが、`Array` ではないため配列メソッドを利用できないことを説明できる
- Arrow Function では `arguments` を参照できず、可変長引数には Rest parameters が推奨されることを説明できる
- 関数の仮引数でオブジェクトの分割代入を使い、渡されたオブジェクトのプロパティを変数として受け取れる
- 関数の仮引数で配列の分割代入を使い、渡された配列の要素を変数として受け取れる
- 関数の仮引数の分割代入を、仮引数を左辺、渡す引数を右辺とした代入に近い構文として説明できる
- 関数名に `()` をつけない場合、関数そのものを値として参照できることを説明できる
- 関数を変数へ代入したり、別の関数の引数として渡したりできることを説明できる
- 関数を値として扱える性質をファーストクラスファンクションとして説明できる
- `const 変数名 = function() { ... };` の形で関数式を書ける
- 関数宣言は文であり、関数式は関数を値として扱う式であることを区別できる
- 関数式では関数名を省略でき、省略された関数を無名関数または匿名関数と呼ぶことを説明できる
- 名前付き関数式の関数名は関数の外からは呼び出せず、関数の中から参照できることを説明できる
- `const 変数名 = () => { ... };` の形で Arrow Function を書ける
- Arrow Function が `=>` を使って無名関数を定義する構文であることを説明できる
- Arrow Function で仮引数が 1 つだけの場合に `()` を省略できる
- Arrow Function の処理が 1 つの式だけの場合に、ブロックと `return` 文を省略し、その式の評価結果を返せる
- Arrow Function は名前をつけられず、`arguments` を参照できず、`new` できないことを本文の範囲で説明できる
- Arrow Function で問題ない場合は Arrow Function を使い、そうでない場合は `function` キーワードを使うという使い分けを説明できる
- 同じ名前の関数宣言を複数書くと、後ろで宣言された関数によって上書きされることを説明できる
- 関数宣言は仮引数の違いでは区別されず、関数名だけで区別されることを説明できる
- 関数宣言と `var` を使った関数式では、同じ名前の関数定義が上書きされることを説明できる
- `const` や `let` では同じ変数名を重複して定義できないため、同じ名前の関数式の定義はエラーになることを説明できる
- 関数の上書きを避けるために、`const` と関数式で関数を定義する方法を選べる
- 引数として渡される関数をコールバック関数と呼び、コールバック関数を引数として使う関数やメソッドを高階関数と呼ぶことを説明できる
- 無名関数をその場で定義して、コールバック関数として渡せる
- `forEach` メソッドが配列の各要素に対してコールバック関数を 1 回ずつ呼び出すことを、本文の範囲で説明できる
- `map` メソッドが配列の各要素をコールバック関数へ渡し、その返り値を新しい配列にすることを、本文の範囲で説明できる
- オブジェクトのプロパティである関数をメソッドと呼ぶことを説明できる
- JavaScript において関数とメソッドに機能的な違いはなく、オブジェクトのプロパティである関数をメソッドと呼び分けていることを説明できる
- オブジェクトリテラルのプロパティに関数式や Arrow Function を代入してメソッドを定義できる
- 既存のオブジェクトのプロパティへ関数を代入してメソッドを定義できる
- `オブジェクト.メソッド名()` の形でメソッドを呼び出せる
- ES2015 のメソッド短縮記法で、オブジェクトリテラル内に `メソッド名() { ... }` と書ける

## 前提として使う既習知識

- `const` や `let` による変数宣言と、変数名に使える名前のルールを使える
- 数値、文字列、真偽値、`undefined`、`null`、配列、オブジェクトの基本的な値を扱える
- 算術演算子、比較演算子、論理 OR 演算子の基本的な結果を読める
- falsy な値と truthy な値の違いを説明できる
- オブジェクトのプロパティアクセスと、配列のインデックスアクセスを読める
- オブジェクトと配列の基本的な分割代入を読める
- `console.log` を使って値を表示できる

## 課題で要求しない知識（必要な場合のみ）

- 関数宣言の巻き上げ（hoisting）の挙動は要求しない
- 関数スコープ、クロージャ、`this` の詳細な挙動は要求しない
- コンストラクタ関数、`new` の詳細、クラス構文、クラスのメソッドは要求しない
- 非同期処理におけるコールバック関数の使い方は要求しない
- DOM API、イベント処理、ブラウザ固有 API は要求しない
- `arguments` を配列へ変換する方法や、`arguments.callee` など本文外の `arguments` の機能は要求しない
- 関数引数以外での Spread 構文や、オブジェクトの Spread 構文は要求しない
- 入れ子の分割代入、プロパティ名の変更、分割代入内のデフォルト値など、本文で扱っていない分割代入の応用は要求しない
- `forEach` と `map` 以外の配列メソッド、または `forEach` と `map` の詳細な仕様は要求しない
- 複雑な再帰アルゴリズムやコールスタックの知識は要求しない

## 課題で扱う場合の注意

- 関数定義を要求する場合は、関数宣言、関数式、Arrow Function、メソッド短縮記法のどれを使ってよいかを課題文で明示する
- Arrow Function を使う課題では、`this` の挙動に依存する問題にしない
- 可変長引数を扱う課題では、原則として Rest parameters を使わせ、`arguments` を使う必要がある場合は課題文で理由を明示する
- デフォルト値を扱う課題では、引数未指定、`undefined`、`null`、空文字列、`0` などの入力で期待する結果を明示する
- `forEach` や `map` を使う課題では、このページで説明されたコールバック関数の受け渡し以上の配列メソッド知識を採点条件にしない
- メソッドを扱う課題では、メソッドがオブジェクトのプロパティである関数だと判断できる範囲に留め、`this` を使ったメソッド設計は要求しない
