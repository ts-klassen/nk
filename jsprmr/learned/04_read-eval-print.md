# 04. 値の評価と表示

## このページで学ぶこと

- JavaScriptコードを評価すると、値、式、変数、変数宣言ごとにどのような結果になるかを扱う
- ブラウザの開発者ツールのWebコンソールで、REPLとしてJavaScriptコードを実行する方法を扱う
- HTMLファイルからJavaScriptファイルを読み込み、ブラウザで実行する方法を扱う
- `console.log`を使って評価結果をコンソールに表示する方法を扱う
- エラーメッセージの基本的な読み方と、構文エラーと実行時エラーの違いを扱う

## 課題で要求してよい知識

- 値や式を評価すると評価結果が返ることを説明できる
- 変数名を評価すると、その変数に代入されている値が評価結果になることを説明できる
- `const x = 1;`のような変数宣言は変数名と値を関連づけるが、変数宣言自体は値を返さないため、REPLでは`undefined`が表示されることを説明できる
- ブラウザの開発者ツールのWebコンソールでJavaScriptコードを1行ずつ評価できることを説明できる
- REPLでは入力したコードを評価し、その評価結果を表示することを説明できる
- REPLでは宣言した変数が実行状態に残るため、同じ`const`変数名を再宣言すると`SyntaxError`になることを説明できる
- REPLの実行状態は、開発者ツールを開いているウェブページをリロードするとリセットされることを説明できる
- HTMLの`<script src="./index.js"></script>`で、同じディレクトリにあるJavaScriptファイルを読み込めることを説明できる
- JavaScriptファイルとして読み込んだコードは、REPLと違って評価結果を自動ではコンソールに表示しないことを説明できる
- `console.log(値)`で、渡した値をコンソールに表示できる
- `console.log(式)`では、先に引数の式を評価してから、その結果をコンソールに表示することを説明できる
- `console.log(変数名)`では、先に変数名を評価してから、その結果をコンソールに表示することを説明できる
- `console.log`を使って途中の値をコンソールに表示し、プリントデバッグの手がかりにできる
- `// => 評価結果`というコメントが、左辺の値を評価した結果またはConsole APIで表示した結果を示すことを読み取れる
- ウェブ版の書籍の実行ボタンでは、実行ごとに新しい環境でコードが実行されるため、REPLのような変数の再定義問題が起きないことを説明できる
- ウェブ版の書籍の実行ボタンでも、値を評価した場合は最後の評価結果が表示され、Console APIにも対応していることを説明できる
- エラーメッセージから、エラーの種類、エラー内容、ファイル名、行番号、列番号を読み取れる
- 構文エラーは文法の間違いがあるコードをパースする際に発生し、プログラムとして実行できないことを説明できる
- `SyntaxError`の例として、関数呼び出しの`)`の不足や`const`のタイプミスから、エラー位置の周辺を確認して原因を探せる
- エラーメッセージと実際の原因が必ずしも一致しない場合があることを説明できる
- 実行時エラーは、構文として正しいコードを実行している最中に発生するエラーであることを説明できる
- `ReferenceError`の例として、存在しない変数を参照したときにエラーになることを説明できる
- 実行時エラーでは、どこまで実行できたかを順番に追い、エラーが発生した行の周辺やエラーメッセージを手がかりに原因を探せる
- エラーの原因を特定し、修正する作業をデバッグと呼ぶことを説明できる
- エラーメッセージを検索したり、MDNのJavaScriptエラーリファレンスを参照したりして原因を調べられることを説明できる

## 前提として使う既習知識

- コメントを使ってコード中に説明を書ける
- `const`で変数を宣言し、値に名前をつけられる
- 同じ名前の`const`変数は再宣言できないことを説明できる
- 文字列や数値のような基本的な値をコード中に書ける

## 課題で要求しない知識（必要な場合のみ）

- Firefox以外のブラウザ固有の開発者ツール操作やエラーメッセージの差分は要求しない
- Console APIのうち、本文で扱っていない`console.log`以外のメソッドは要求しない
- DOM APIを使った画面操作は要求しない
- `try...catch`などの例外処理は要求しない
- `TypeError`など、本文で名前だけ触れた実行時エラーの詳細な発生条件は要求しない
- Node.jsでの実行方法やパッケージ管理の知識は要求しない

## 課題で扱う場合の注意

- ブラウザや実行環境によってエラーメッセージの文言は異なるため、完全一致ではなくエラーの種類、位置情報、原因の説明を評価する
- HTMLファイルとJavaScriptファイルを使う課題では、文字コードをUTF-8、改行コードをLFとして保存する前提を明示する
- ローカルHTMLファイルをブラウザで開く課題では、`file:///`から始まるローカルファイルとして実行する範囲に留める
- REPLで同じ変数名を繰り返し使う課題では、必要に応じてページのリロードで実行状態をリセットすることを案内する
- `console.log`で表示する値と、式や変数を評価しただけの結果を区別して採点する
