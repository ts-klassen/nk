# 商品カードの更新

## 問題

`input` には、次のプロパティを持つオブジェクトが入っています。

- `base`: もとの商品カードを表すオブジェクト
- `patch`: 更新したいプロパティだけを持つオブジェクト
- `required`: 必ず確認するプロパティ名の配列
- `remove`: 削除したいプロパティ名を表す文字列

次の手順で作った新しいオブジェクトを `output` に代入してください。

1. `base` と `patch` をマージした新しいオブジェクトを作る。同じプロパティ名がある場合は、`patch` の値を使う。
2. その新しいオブジェクトから、`remove` の値と同じ名前のプロパティを削除する。
3. `required` の各プロパティ名について、削除後の新しいオブジェクトがそのプロパティを持っているか確認する。
4. 持っていないプロパティだけを追加し、値を `"未設定"` にする。

`base` と `patch` は変更しないでください。

プロパティの存在確認では、値が `0`、`false`、`null` であっても、そのプロパティ自体があれば「持っている」と扱います。

## 入力

`input` には、`base`、`patch`、`required`、`remove` を持つオブジェクトが入っています。

`required` に同じプロパティ名は入りません。

## 出力

`output` に、更新後の商品カードを表すオブジェクトを代入してください。

## 例

`input`:

```json
{
  "base": {
    "name": "ノート",
    "price": 120,
    "stock": 5,
    "label": "old"
  },
  "patch": {
    "price": 150,
    "sale": true
  },
  "required": ["name", "price", "stock", "sale", "label"],
  "remove": "label"
}
```

期待される `output`:

```json
{
  "name": "ノート",
  "price": 150,
  "stock": 5,
  "sale": true,
  "label": "未設定"
}
```
