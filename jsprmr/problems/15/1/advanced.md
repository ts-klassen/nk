# 公開する記事名

## 問題

`input.paths` には、記事ファイルのパスを表す文字列の配列が入っています。
`input.extension` には、対象にする拡張子を表す文字列が入っています。

公開する記事だけを選び、記事名を整形した文字列を作ってください。

各パスについて、次の手順で判定と整形をします。

1. 最後の `/` より後ろをファイル名として取り出します。`/` がない場合は、パス全体をファイル名とします。
2. ファイル名が `input.extension` で終わり、かつ `_` で始まらない場合だけ公開対象にします。
3. 公開対象のファイル名から、末尾の `input.extension` を取り除きます。
4. 残った文字列に含まれるすべての `-` を半角スペースに置き換えます。

公開対象の記事名を、元の配列の順番のまま `", "` で結合してください。
公開対象が 1 つもない場合は、空文字列にしてください。

## 入力

`input` には次の形式のオブジェクトが入っています。

```js
{
  paths: パス文字列の配列,
  extension: 対象にする拡張子の文字列
}
```

`input.extension` は空文字列ではなく、`.` から始まります。

## 出力

`output` に、整形した記事名を結合した文字列を代入してください。

## 例

`input`:

```json
{
  "paths": [
    "posts/js-basics.md",
    "posts/_draft.md",
    "archive/string-methods.md",
    "README.txt"
  ],
  "extension": ".md"
}
```

期待される `output`:

```json
"js basics, string methods"
```
