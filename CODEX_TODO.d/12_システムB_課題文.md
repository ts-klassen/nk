# システム B の課題文を作る

## 目的

受講生が用例採集システムを一から実装するための課題文を作成する。

## 課題の前提

- 受講生はシステム A の教材を学習済み。
- システム B は一から実装する。
- SQL と API 仕様書は提供される。
- テストは全て公開される。
- 非公開テストはない。
- 評価基準は非公表とする。
- 受講生向けセルフチェックリストは作らない。

## 課題で実装するもの

- Express アプリ
- TypeScript 設定
- MySQL 接続
- DB 初期化手順
- 共通エラーレスポンス
- 入力バリデーション
- Basic 認証
- ユーザー登録
- 用語 CRUD
- 用例 CRUD
- 用例の所有者チェック
- 用語削除時の子データあり `409 Conflict`

## 課題文に書くこと

- 課題の目的
- 提出形式
- 起動方法
- テスト実行方法
- 提供物
- 実装すべき API 一覧
- 認証仕様
- エラー仕様
- 注意事項

## 課題文に書かないこと

- 評価基準
- セルフチェックリスト
- 実装の答え
- 非公開テストを前提にした説明
