# 共通認証仕様を定義する

## 目的

ユーザー登録、Basic 認証、Argon2 によるパスワードハッシュ化の仕様を定義する。

## 決定済み仕様

- Basic 認証を使う。
- JWT は使わない。
- セッション管理はしない。
- Basic 認証の ID は `username` とする。
- パスワードは Argon2 でハッシュ化して保存する。
- `POST /users` は公開 API とする。
- ユーザー更新 API と削除 API は作らない。
- `username` は半角英数字と `_` のみ、3 文字以上 32 文字以下とする。
- `password` は 8 文字以上 72 文字以下とする。
- `username` は一意とする。
- `username` 重複時は `409 Conflict` と `CONFLICT` を返す。
- 認証失敗時は `401 Unauthorized` と `UNAUTHORIZED` を返す。
- 認証失敗時は `WWW-Authenticate: Basic realm="backend-training"` ヘッダを返す。
- 認証済みだが権限がない場合は `403 Forbidden` と `FORBIDDEN` を返す。
- 認証失敗時の message は `認証が必要です` とする。
- Argon2 はライブラリのデフォルトパラメータを使う。教材ではパスワードを平文保存せず、ハッシュ化して検証する流れまでを扱い、Argon2 パラメータ調整は扱わない。

## ユーザー登録リクエスト

```json
{
  "username": "alice",
  "password": "password123"
}
```

## ユーザー登録レスポンス

```json
{
  "id": 1,
  "username": "alice",
  "createdAt": "2026-04-28T11:39:55+09:00",
  "updatedAt": "2026-04-28T11:39:55+09:00"
}
```
