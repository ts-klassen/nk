# システム A の任意テスト教材を作る

## 目的

時間がある受講生向けに、システム A の API 結合テストを書く教材を用意する。

## 位置づけ

この作業は必須ではない。時間があれば扱う。

## 方針

- Mocha / Chai を使う。
- BDD スタイルで書く。
- DB 込みの API 結合テストにする。
- 単体テストは書かない。
- 正常系と異常系を扱う。
- テスト実行ごとに DB を初期化する。

## テスト対象候補

- ユーザー登録成功
- `username` 重複時の `409 Conflict`
- 書籍作成成功
- ISBN 重複時の `409 Conflict`
- 書籍一覧の `limit` / `offset`
- 読書メモ作成成功
- 認証なし読書メモ作成時の `401 Unauthorized`
- 他ユーザーの読書メモ参照時の `403 Forbidden`
- 読書メモあり書籍削除時の `409 Conflict`
