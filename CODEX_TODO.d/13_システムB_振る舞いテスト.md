# システム B の振る舞いテストを作る

## 目的

用例採集システムの振る舞いテストを作成する。

## 方針

- Mocha / Chai を使う。
- BDD スタイルで書く。
- DB 込みの API 結合テストにする。
- 単体テストは書かない。
- 正常系と異常系を含める。
- テスト実行ごとに DB を初期化する。

## テスト対象候補

- ユーザー登録成功
- `username` 重複時の `409 Conflict`
- Basic 認証失敗時の `401 Unauthorized`
- `WWW-Authenticate` ヘッダ
- 用語作成成功
- 用語一覧の `limit` / `offset` / `total`
- 用語詳細取得成功
- 用語更新成功
- 用語削除成功
- `term` 重複時の `409 Conflict`
- 用例作成成功
- 認証なし用例作成時の `401 Unauthorized`
- 用語に紐付く用例一覧取得
- 用例詳細取得
- 他ユーザーの用例更新時の `403 Forbidden`
- 他ユーザーの用例削除時の `403 Forbidden`
- 用例あり用語削除時の `409 Conflict`
- 存在しないリソースの `404 Not Found`
- バリデーションエラーの `400 Bad Request`
