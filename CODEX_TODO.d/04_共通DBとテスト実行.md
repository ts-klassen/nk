# 共通 DB・テスト実行方針を定義する

## 目的

MySQL、Docker Compose、SQL ファイル、テスト実行時の DB 初期化方針を定義する。

## 決定済み仕様

- MySQL を使う。
- Docker Compose で起動する。
- ORM は使わない。
- `mysql2` で SQL を直接実行する。
- migration ツールは使わない。
- `schema.sql` と必要に応じて `seed.sql` を使う。
- 主キーは `AUTO_INCREMENT` とする。
- テストは DB 込みの API 結合テストとする。
- 単体テストは書かない。
- テスト実行ごとに DB を初期化する。
- システム B のテストは全て公開する。

## テスト実行方針

テスト開始時に DB を初期化する。

```text
DROP DATABASE
CREATE DATABASE
schema.sql 実行
seed.sql 実行
テスト実行
```
