# 5 日間の研修構成を整理する

## 目的

5 日間の自習教材と課題レビューの流れを整理する。

## 前提

- 受講生は TypeScript をフロントエンド学習で習得済み。
- バックエンドと SQL は未経験寄り。
- JOIN など複雑な SQL は別枠で学習する。
- 講師・メンターは質問対応とデイリーミーティングを行う。

## 構成案

### 1 日目

- REST API の基本
- Express アプリの構成
- JSON リクエストとレスポンス
- 共通エラーレスポンス
- 書籍の読み取り API

### 2 日目

- MySQL 連携
- `mysql2` による SQL 実行
- `schema.sql`
- 書籍 CRUD
- `limit` / `offset`

### 3 日目

- 入力バリデーション
- `VALIDATION_ERROR`
- `NOT_FOUND`
- `CONFLICT`
- 子データあり削除時の `409 Conflict`

### 4 日目

- ユーザー登録
- Argon2
- Basic 認証
- 読書メモ CRUD
- 所有者チェック
- `401 Unauthorized`
- `403 Forbidden`

### 5 日目

- システム B の課題実装
- 公開テスト実行
- 質問対応
- 課題レビュー

## 発展課題
