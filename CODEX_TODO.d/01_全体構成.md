# 全体構成を設計する

## 目的

教材、システム A、システム B、DB、テスト、実行手順の置き場所を決める。

## 入力

- 確定済み仕様
- 自習教材と課題レビューという研修形式

## 作るもの

- ディレクトリ構成案
- npm scripts 案
- Docker Compose 構成案
- 環境変数一覧
- README に載せる実行手順の範囲

## 方針

- 教材本文、システム A、システム B を混同しない構成にする。
- システム A は教材で説明する完成実装として扱う。
- システム B は受講生が一から実装する課題として扱う。
- フロントエンドとの型共有やコード共有はしない。

## 候補構成

```text
docs/
  system-a/
  system-b/
system-a/
  src/
  sql/
  test/
system-b/
  src/
  sql/
  test/
```
